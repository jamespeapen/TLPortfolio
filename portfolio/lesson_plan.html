Data Analysis and Bioinformatics

Rmarkdown

Feb 13, 2023

Session overview

One of the course learning outcomes was to design and teach a lesson plan. I chose class and lesson plan because this is the type of class I can see myself teaching and it was helpful to think about teaching others with a structure what I taught myself in a less structured way

In this class, students will be introduced to rmarkdown. You will also learn about the final project requirements and its submission format as an rmarkdown report. The class will introduce markdown and its syntax, how rmarkdown extends on markdown, and how to write an rmarkdown documents that can generate PDF, HTML, and Word reports according to your requirements. We will learn how to include reproducible code, analysis and results.

Learning outcomes

After this lesson, you will be able to

  1. Create a new Rmarkdown file in RStudio and knit it to PDF, HTML, and Word documents.

  2. Use R and bash code chunks to run statistical analyses and generate plots

  3. Configure how code chunks and plots are knit to the final output document.

Resources

All resources for the lesson are on the class Github repository:

https://github.com/Bioinformatics-Training-VAI/2023-data-sci-bioinfo/tree/rmarkdown/session5_rmarkdown

Only accessible for students in the class

There is a example.rmd which students will download for the practice parts of class.

Lesson plan

I overestimated some of the timings in the plan and the class finished much earlier than thought it would. However, this meant students had time to work on the homework and ask for help and troubleshooting while I was with them.

To follow my problem solving as a learning goal principle, I tried to use the instructors description of the final project as the problem to be solved and the what they learn about Rmarkdown in class as the solution to that problem, with opportunities in class to simulate a final project.


This would be a programming variant of think-pair-share where pair-programming helps students think about another persons problem and apply their knowledge in a slightly different situation. It also helps the person currently explaining their problem to have a moment of metacognition as they think about their learning and try to verbalize the problem they have.

Context and relevance

This lesson is part of the Data Analysis and Bioinformatics class at VAIGS. The course goal is to introduce first-year PhD students to common bioinformatic tools and teach them how to use them during in their work. Before this lesson, students would have learned how to navigate a linux filesystem and manipulate files, use git and github to share and version control files, and use R for basic data input and statistical analysis. For this lesson, students will need to remember the R code they learned in the previous two lessons. During this lesson they will learn how the analysis done with those methods can be integrated into one shareable, version controlled report that includes both their code and their analysis. Subsequent lessons will have more involved analyses that would require students to maintain reproducible reports using Rmarkdown and the course will end with them presenting their project as the output of an Rmarkdown document with properly formatted code and results.

Pre-teaching reflection

During this lesson, the hardest thing for students to work with would be the move from writing prose in a what-you-see-is-what-you-get editor to one where they have to set formatting and layout using symbols and code. There are also a lot of configuration options that may overwhelm them. As an instructor, I would need to make sure they are all on the same page, and work with any problems that might result from differences in the laptops they have or their prior skillset. I would also have to make sure they understand the configuration options they need, but also let them know that although there are a lot of options, they don’t need to know or understand all of them in this lesson. They can be confident that what they learn here can be useful to them and help them learn more.

Post-teaching reflection

The course instructors introduced the final project requirements - which is the summative assessment for the class. However, they did it a little differently than I expected from my time in the class and did not go into a lot of detail on the datasets required or the final presentation format. I tried to incorporate a bit of that in my introduction.

The introduction went well, but the first practice/test of their ability to run the code did not go smoothly since there were differences in their computer hardware and R installations that gave some trouble. It was also clear that not everyone had done the pre-class assignment fully. This did not take too long since I was able to give them alternative ways to follow along in class.

During the class, some of the students showed that they had some experience with the subject and had informed questions, while others were very quiet, even when asked whether they had caught up to what I had asked them to do.

I had an hour at the end where they were working on their formative assessment. The goal was to get them to practice working with Markdown and generate a short, but full report on a small dataset. I’m glad I had the full time, since there was a wider range of skill than I expected and this allowed me to walk around the class as they worked and help as needed. There were some students who asked for help only as I came around. The slower students were able to ask for help to complete the work, while the faster ones completed the work and had time to explore other configuration options for their reports.

I think I could have broken down the material and added more to make it more granular - they did not know some things I thought they would. Some of it was outside my control and from the previous classes that I only knew the title of - I didn’t attend the class and see exactly what they learned and how proficient they got. It seemed like they learned some things, but did not have much time to practice them.
