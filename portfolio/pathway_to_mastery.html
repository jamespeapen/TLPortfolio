Pathway to mastery

Course description

Modern scientific research is driven by computing power. Using this power effectively is key to driving good research and results. This course surveys the scientific computing landscape and the tools available for scientists that will help you as you solve research problems. You will learn to use basic scientific computing tools like Linux and the python programming language to solve scientific problems. We will also learn skills like filesystem organization, text manipulation, and choosing data structures and algorithms that are best suited to our research needs. You will get to Practice computing skills on data and research questions that are of interest to you – you could even bring data from other classes that you need to analyze. Instead of tests, you will learn to read documentation and effectively use the internet to find solutions to your programming problems and demonstrate your mastery of the class learning goals. After completing the course, you will have the skills to automate parts of your research workflow, increase the efficiency with which you process and analyze data, collaborate effectively on data driven projects and fix issues as they arise, and write maintainable code.

Course learning objectives

No. Key impacts Draft Statements CLOs
1 Regular expressions Write regular expressions to capture strings of interest. Create regular expressions that capture and/or replace specific patterns like filenames, data columns, and command line utility outputs using sed.
Write expressions to replace strings and patterns.
Patterns of interest should be scientifically relevant, students don’t have to be able to capture any pattern.
2 Appropriate data structures and algorithms Write scripts to process data with Python. Justify your data structure and algorithm choices with respect to how they are suited to your requirements and use cases and your error handling choices with respect to user experience.
User interaction Write scripts to automate workflows and tasks.
Version control Maintain a record of the development using Git.

1. Connect

Think and share times when you needed to pull out a portion of text of a certain pattern. Did your word processor’s find and replace help them in cases where the text was just slightly different?

2. Acquire

Regular expressions as an extension of the find and replace idea using general patterns.

3. Practice

In class group: Capture word patterns from text using regex101.com

4. Feedback

Pre-class assessment of what student know about data types

5. Acquire

Demo: Learn about the different string types in text and the symbols used to represent them generally.

6. Practice

In groups: Capture given patterns of letters and numbers from a body of text.

7. Feedback

What do you think you can effectively capture so far? Are there patterns that are difficult or frustrating to write and why?

8. Assess

Formative checkpoint: Capture patterns with letters and numbers

9. Acquire

Class demo: Combining the data type symbols with wildcards to capture repeated patterns within the main patterns.

10. Practice

Capture more complex of letters and numbers from a body of text.

11. Assess

Formative checkpoint: List the symbols and wildcards to represent text and repeats.

12. Connect

From your research (or your other professors) bring data that has patterns that need extracting. How have you/they done it?

13. Practice

Formative checkpoint: Using the data brought to class capture the patterns of interest from them.

14. Feedback

How did you find the real-world data? Do you think what you’ve learned here can be used or is the data to complex?
